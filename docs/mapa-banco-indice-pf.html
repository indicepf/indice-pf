Índice PF — Mapa do Banco de Dados

Tabelas, chaves, relações, políticas de acesso (RLS), funções/gatilhos e onde cada tabela é usada no app.

Fonte: reconstruído das migrações (Migration Supabase/*.sql) e do código (lib/queries.ts, componentes e scripts Python). As tabelas snapshots, precos, resultados_brutos e a view historico_precos vêm do schema original (as migrações desta pasta não as criam) — suas colunas foram inferidas do uso no código. O schema vivo no Supabase pode divergir se houver alteração manual; para 100% de fidelidade, rode uma introspecção (information_schema) no painel.

4. Bloco C — Autenticação e segurança

profiles

ColunaTipoPapel
iduuidPK e FK → auth.users(id) (1:1 com o usuário)
nome, telefone, regiao, sexo, data_nascimento—perfil (telefone é o lead)
cpf, chave_pix, consentimento_cpf_em—dados de recompensa (LGPD)
is_admin, is_superboolpapéis de acesso

RLS: cada um lê/edita o próprio (perfil_select/perfil_update); admin lê todos (perfil_admin_select). Criada automaticamente no cadastro por handle_new_user.

Usado em: getProfile, useAuth, isAdmin/isSuper, getPerfis, nomesPorId, Auth, /perfil, Painel · RPC super_editar_perfil. Base das funções eh_admin()/eh_super().

TabelaConteúdoAcesso
